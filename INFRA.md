
1. **The Ingress Node (Edge Router)**
* **Component:** Traefik API Gateway.


* **Role:** The single entry point into the local cluster. Replaces local Docker Compose labels with Ingress resources in production.


* **Connections:** Receives all client HTTP and WebSocket traffic. Routes `api.localhost` to the Core API and `tracking.localhost` to the Tracking API.



2. **The Core Services Cluster (High Consistency)**
* **Component:** Core API (Hono/Bun).


* **Role:** Handles business domain logic, transactions, order creation, and database writes.


* **Component:** PostgreSQL Database.


* **Role:** Relational state storage accessed via Drizzle ORM.


* **Connections:** The Core API maintains a resilient connection pool to PostgreSQL.




3. **The Tracking Services Cluster (High Frequency)**
* **Component:** Tracking API (Go).


* **Role:** Upgrades HTTP to WebSockets, managing active courier sessions.


* **Component:** Redis.


* **Role:** In-memory cache to store high-frequency courier GPS coordinates.


* **Connections:** The Tracking API maintains a fast read/write connection to Redis.




4. **The Asynchronous Bridge (Event Bus)**
* **Component:** RabbitMQ.


* **Role:** Decouples the Core and Tracking services.
* **Connections:** The Core API acts as a *Publisher* (e.g., publishing an "Order Ready for Delivery" event). The Tracking API acts as a *Consumer*, reading these events to trigger WebSocket broadcasts to clients.




5. **The Hybrid Cloud Pipeline (AWS)**
* **Component:** AWS S3.


* **Role:** Receives raw image uploads directly from the client via Pre-signed URLs generated by the Core API.


* **Component:** AWS Lambda (Node.js/Python).


* **Role:** Serverless function triggered automatically by S3 events to compress images.


* **Connections:** Client uploads direct to S3. S3 triggers Lambda. Lambda fires a secure Webhook callback over the internet back to the Traefik Ingress -> Core API to save the final image URL in PostgreSQL.
